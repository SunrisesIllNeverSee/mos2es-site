## 1. What is SEO vs GEO vs AEO?

| Layer | What it optimizes for | Example engines |
|-------|----------------------|-----------------|
| **SEO** | Classic search rankings (Google, Bing) | Google Search, Bing |
| **GEO** | Generative engine optimization — getting cited by AI | ChatGPT, Perplexity, Claude, Google AI Overviews |
| **AEO** | Answer engine optimization — appearing in answer boxes | Google Featured Snippets, voice assistants |

The key insight: **the same content serves all three**, but the structure
matters. AI engines extract by heading, lift the first 1-2 sentences under
each heading, and prefer tables and step-by-steps over prose. Classic search
cares about backlinks and title/meta. Both care about freshness.

## Phase 1 — Fix the OG Image

### The problem
The OG image was an SVG file (`/og.svg`). Most platforms (Slack, Twitter,
Discord, iMessage) don't render SVG — they show a blank or broken preview.

### What we did
Replaced it with a real 1200×630 PNG generated by `next/og` (Satori). On
a static HTML site, you'd just create a PNG in Photoshop/Figma/Canva and
reference it in your `<meta>` tags.

### How to do it on mos2es.com
1. Create a 1200×630 PNG banner (dark background, MO§ES logo, tagline)
2. Save as `/img/og.png`
3. Add to every page's `<head>`:
```html
<meta property="og:image" content="https://mos2es.com/img/og.png">
<meta property="og:image:width" content="1200">
<meta property="og:image:height" content="630">
<meta name="twitter:card" content="summary_large_image">
<meta name="twitter:image" content="https://mos2es.com/img/og.png">
```
4. Verify at https://www.opengraph.xyz

### Acceptance
Sharing any mos2es.com URL in Slack/Twitter shows a real card with the
banner image, title, and description.

## Phase 2 — JSON-LD Structured Data

### Why this is the biggest lever
JSON-LD structured data tells search engines and AI engines exactly what
your content is. AI engines (ChatGPT, Perplexity, Claude) use it to decide
what to cite. Google uses it for rich results.

### What we shipped on signalaf.com
- **Organization** + **WebSite** schema on every page (in layout.tsx)
- **ItemList** on leaderboard pages (list of ranked operators)
- **ProfilePage** on operator profiles
- **BreadcrumbList** on wiki pages
- **DefinedTerm** on glossary/concept pages

### How to do it on mos2es.com
You already have Organization schema on the homepage — good. Add:

**WebSite schema** (on every page):
```html
<script type="application/ld+json">
{
  "@context": "https://schema.org",
  "@type": "WebSite",
  "@id": "https://mos2es.com/#website",
  "name": "MO§ES",
  "url": "https://mos2es.com",
  "publisher": { "@id": "https://mos2es.com/#org" }
}
</script>
```

**DefinedTerm** on papers.html (for each concept):
```html
<script type="application/ld+json">
{
  "@context": "https://schema.org",
  "@type": "DefinedTerm",
  "name": "Conservation Law of Commitment",
  "description": "C(T(S)) ≈ C(S) with enforcement; C(T(S)) < C(S) without it.",
  "url": "https://mos2es.com/papers.html",
  "inDefinedTermSet": {
    "@type": "DefinedTermSet",
    "name": "MO§ES Concepts",
    "url": "https://mos2es.com"
  }
}
</script>
```

**BreadcrumbList** on sub-pages:
```html
<script type="application/ld+json">
{
  "@context": "https://schema.org",
  "@type": "BreadcrumbList",
  "itemListElement": [
    { "@type": "ListItem", "position": 1, "name": "Home", "item": "https://mos2es.com/" },
    { "@type": "ListItem", "position": 2, "name": "Architecture", "item": "https://mos2es.com/architecture.html" }
  ]
}
</script>
```

**FAQPage** (if you add Q&A content):
```html
<script type="application/ld+json">
{
  "@context": "https://schema.org",
  "@type": "FAQPage",
  "mainEntity": [
    {
      "@type": "Question",
      "name": "What is the Conservation Law of Commitment?",
      "acceptedAnswer": {
        "@type": "Answer",
        "text": "C(T(S)) ≈ C(S) with enforcement; C(T(S)) < C(S) without it. The law states that semantic commitment is conserved under transformation only when governance is enforced."
      }
    }
  ]
}
</script>
```

### Validate
- https://validator.schema.org — paste each page URL
- https://search.google.com/test/rich-results — paste deployed URLs

### Acceptance
Every page type passes the Schema validator with 0 errors.

## Phase 3 — llms.txt

### What it is
A plain-text file at `/llms.txt` that tells AI crawlers what your site is
and where the canonical content lives. It's the GEO equivalent of
robots.txt — not a standard, but increasingly respected by AI engines.

### What we shipped
A route at `/llms.txt` with:
- Site name + tagline
- Core pages (with links)
- Concepts/definitions (with links)
- Tooling (npm, GitHub)
- Academic foundation (DOIs)

You already have this on mos2es.com — it's good. Make sure to:
1. Add any new pages to llms.txt manually (it's not auto-generated)
2. Verify all links resolve (no 404s)
3. Consider a `/llms-full.txt` that inlines full definitions for maximum
   citation coverage

### Verify
```bash
curl -s https://mos2es.com/llms.txt | wc -l           # line count
curl -s https://mos2es.com/llms.txt | grep -c "http"  # link count
# Check for 404s in llms.txt links:
curl -s https://mos2es.com/llms.txt | grep -oE 'https://mos2es.com[^ )]+' | while read url; do
  code=$(curl -s --max-time 5 -o /dev/null -w "%{http_code}" "$url")
  [ "$code" != "200" ] && echo "$code $url"
done
```

## Phase 7 — Screaming Frog Site Audit

### What Screaming Frog is
A desktop SEO crawler (free tier: 500 URLs). It crawls your entire site
like Googlebot does, then exports reports on:
- Broken links (404s)
- Redirect chains
- Orphan pages (in sitemap but not linked internally)
- Missing/duplicate title tags and meta descriptions
- Missing H1/H2 tags
- Duplicate content
- Missing image size attributes
- Security headers
- Internal linking structure (crawl depth, inlink count)

### Download
https://www.screamingfrog.co.uk/seo-spider/

### How we ran it on signalaf.com

**Crawl 1 (baseline):**
1. Opened Screaming Frog, entered `https://signalaf.com`
2. Clicked Start
3. Waited ~1 minute (127 URLs crawled)
4. Exported:
   - Visualisations → Force-Directed Crawl Diagram (SVG)
   - Visualisations → Tree Graph → Site Structure (SVG)
   - File → Export → All Internal Links (CSV)
   - File → Export → All External Links (CSV)
   - Reports → Orphan Pages (CSV)
   - Reports → Redirects (CSV)
   - Overview → Issues (CSV)

**What crawl 1 found (30 issues, grade B-):**
- 1 HIGH: broken link (`/submit` → 404)
- 46 unnecessary redirects (`/leaderboard` → `/board/all`)
- 4 security headers missing on 100% of pages
- 34 meta descriptions over 155 chars
- 8 page titles over 60 chars
- 5 low-content pages
- 18 duplicate H2s
- 3 images missing size attributes
- 9 sitemap orphan pages (in sitemap but not linked from any page)

**The fix campaign (5 commits, ~4 hours):**

| Fix | What we did | Result |
|-----|-------------|--------|
| Broken link | Removed `/submit` link + sitemap entry | 404 eliminated |
| Redirects | Changed all `/leaderboard` links to `/board/all` | 46 redirects → 0 |
| Internal linking | Added footer Topics column + topic hub cross-links + 27 SEO pages linked to hubs | Avg inlinks/page 18.1 → 27.0 (+49%) |
| Security headers | Added CSP, X-Frame-Options, X-Content-Type-Options, Referrer-Policy | 100% missing → 100% present |
| Meta descriptions | Shortened all 37 pages to ≤155 chars | 34 over → 8 over |
| Page titles | Shortened 11 long titles to 30-60 chars | 8 over → 0 over |
| Low content | Added intro content to thin pages | 5 → 3 |
| H1/H2 | Fixed dynamic board H1, unique metrics H2s | 18 dup → 17 dup |
| Image sizes | Added width/height to 7 images | 3 missing → 0 |

**Crawl 2 (post-fix):**
- 30 issues → 23 issues (7 fully eliminated)
- 487 flagged URLs → 174 (64% reduction)
- Grade B- → A-
- 0 broken links, 0 internal redirects, 0 missing security headers

### How to run it on mos2es.com

1. Download Screaming Frog SEO Spider (free, 500 URL limit)
2. Open it, enter `https://mos2es.com` as the URL
3. Click Start
4. Wait for the crawl (11 pages, under 30 seconds)
5. Export the same files listed above
6. Drop the exports where your AI agent can read them

**What to look for:**
- **404s** — fix or remove the broken links
- **Redirect chains** — flatten them (A → C, not A → B → C)
- **Orphan pages** — pages in your sitemap but not linked from any page
- **Missing meta descriptions** — every page needs one, ≤155 chars
- **Long titles** — keep ≤60 chars
- **Missing H1** — every page needs exactly one H1
- **Duplicate H2s** — each page should have unique H2s
- **Security headers** — CSP, X-Frame-Options, X-Content-Type-Options, Referrer-Policy
- **Crawl depth** — no page should be more than 3 clicks from the homepage
- **Images missing size** — add width/height attributes

**After the crawl:**
1. Generate a fix list (prioritize: broken links > redirects > security > meta > titles > H1/H2 > images)
2. Fix in order of priority
3. Re-crawl to verify (compare crawl 1 vs crawl 2)
4. Track issue count over time — it should never INCREASE

### Screaming Frog features reference

| Feature | Use it for |
|---------|-----------|
| Find Broken Links | 404s and server errors |
| Audit Redirects | Redirect chains and loops |
| Analyse Page Titles & Meta | Missing/duplicate/too-long titles and descriptions |
| Discover Duplicate Content | Exact and partial duplicates |
| Extract Data with XPath | Pull custom data from HTML |
| Review Robots & Directives | noindex/nofollow/canonical issues |
| Generate XML Sitemaps | Compare generated vs your sitemap |
| Integrate with GA, GSC & PSI | Pull performance data per URL |
| Crawl JavaScript Websites | Render JS (important for SPAs) |
| Visualise Site Architecture | Crawl diagram + tree graph |
| Schedule Audits | Recurring crawls — set monthly |
| Compare Crawls | Track SEO issues over time |

## Post-Audit — Indexing Push

After fixing the issues, we pushed the site to search engines:

### Google Search Console
1. Submit `sitemap.xml`
2. Request indexing on key pages
3. Use the Indexing API for bulk URL submission

### IndexNow (Bing/Yandex/Naver/Seznam)
Instant indexing — free, no API key needed (just a key file on your domain):

1. Generate a key: `openssl rand -hex 16`
2. Save as `https://mos2es.com/<key>.txt` (file contains just the key)
3. Push URLs:
```bash
curl -X POST "https://api.indexnow.org/IndexNow" \
  -H "Content-Type: application/json" \
  -d '{
    "host": "mos2es.com",
    "key": "<your-key>",
    "keyLocation": "https://mos2es.com/<your-key>.txt",
    "urlList": [
      "https://mos2es.com/",
      "https://mos2es.com/papers.html",
      "https://mos2es.com/architecture.html"
    ]
  }'
```

### On mos2es.com
You already have an IndexNow key file (`3cb9dad60ebc43248d4ec58b2d9b4aca.txt`).
Push all 11 URLs to IndexNow after any content change.

### Weekly: Citation Query Test
Run 10-15 target queries in **incognito mode** across ChatGPT, Perplexity,
Claude, and Google AI Overviews. Record: mentioned? cited? which competitor
cited instead?

Example queries for MO§ES:
```
1. what is conservation law of commitment
2. AI governance framework
3. multi-agent governance protocol
4. commitment conservation AI
5. semantic meaning preservation AI
6. SHA-256 audit trail AI agents
7. lineage bound artifacts
8. recursive compression AI governance
9. governance vacuum AI
10. sovereign signal governance
```

### Monthly: AEO Citation Audit
For each target query, record in a spreadsheet:
- Date
- Engine (ChatGPT / Perplexity / Claude / Google)
- Query
- Mentioned? (yes/no)
- Cited? (yes/no)
- Which competitor cited instead?

Run monthly, compare trends. **Perplexity is the citation leader** — if
you're not showing up there, your content structure needs work.

### Quarterly: Screaming Frog Crawl
Re-run the crawl, compare to previous. Track that issue count doesn't
increase. Look for:
- New 404s (from route changes)
- New redirect chains
- New orphan pages
- Duplicate titles/meta (from new pages)
- Missing H1/H2 on new pages

## What to Expect: Timeline and Results

### What happened on signalaf.com

| Timeframe | Result |
|-----------|--------|
| **Week 1** | Pages indexed by Google (after GSC + IndexNow push) |
| **Week 2** | Pages indexed by Bing (IndexNow is fast) |
| **Week 2-4** | First GSC impressions start appearing |
| **Month 1** | ~100 impressions, 0 clicks (baseline) |
| **Month 2-3** | Impressions grow as content pages get crawled |
| **Month 3-6** | First AI citations (Perplexity usually first) |
| **Month 6+** | Citation share grows if content is refreshed |

### Realistic expectations for mos2es.com

- **11 pages is small.** The biggest ROI is adding concept/guide pages
  (Phase 6). Each new page is one more search intent you capture.
- **AI citations take time.** Don't expect Perplexity to cite you in week 1.
  The content needs to be indexed first, then the AI engines need to crawl
  it, then they need to choose to cite it. 2-6 months is normal.
- **GSC is free and essential.** Set up Google Search Console immediately.
  Submit your sitemap. Check it weekly.
- **IndexNow is the fastest path to Bing indexing.** Push URLs after every
  content change.
- **The feedback loop is what most teams skip.** Every finding needs an
  action item. Don't just log findings — act on them.

## What NOT to Do

1. **Don't block AI crawlers** — being in the training corpus is the moat
2. **Don't audit once and never re-run** — citation share shifts weekly
3. **Don't trust a single LLM snapshot** — run queries 2-3 times (LLMs have variance)
4. **Don't create new content before refreshing existing** — 70/30 rule (70% refresh, 30% new)
5. **Don't push already-indexed URLs to GSC** — wastes API quota
6. **Don't change robots.txt without verifying AI bot access after**
7. **Don't forget llms.txt is manual** — new pages need to be added by hand
8. **Don't ignore Perplexity** — it's the most citation-heavy engine
9. **Don't skip the feedback loop** — every finding needs an action item
10. **Don't use SVG for OG images** — most platforms don't render them
