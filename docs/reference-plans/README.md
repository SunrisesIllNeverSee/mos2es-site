# Reference Plans — signalaf.com SEO/GEO/AEO Implementation

These are the original plan and audit documents from the signalaf.com
implementation. They're the source material that the two teaching docs
(`SEO_GEO_AEO_PLAYBOOK.md` and `SEO_BEEFUP_AND_ACADEMIC_GEO.md` in the
repo root) were built from.

## What's in here

| File | What it is |
|------|-----------|
| `SEO_GEO_PLAN.md` | The original 7-phase SEO/GEO implementation plan (OG images, JSON-LD, llms.txt, dynamic OG, npm/GitHub, content layer, Screaming Frog). All phases shipped. |
| `SEO_AEO_GEO_MAINTENANCE.md` | The upkeep playbook — weekly/monthly/quarterly/annual cadence. Citation tracking, Screaming Frog re-crawls, content decay, llms.txt audits. |
| `ACADEMIC_GEO_IMPLEMENTATION.md` | The signalaf.com-side changes to connect the live product to the academic foundation (Organization sameAs, ScholarlyArticle, /science page, llms.txt expansion, Dataset citation field). |
| `ACADEMIC_GEO_MAPPING_PROMPT.md` | The full ecosystem mapping prompt — audits 40 GitHub repos, Zenodo deposits, live sites, and builds a unified linking strategy so every surface points to every other. |
| `SITE_AUDIT_signalaf_2026-07-09.md` | Screaming Frog crawl 1 analysis — full internal link map, Mermaid diagram, all 30 issues found. |
| `FIX_PLAN.md` | The prioritized fix plan — phases A through H, with exact file changes and effort estimates. |
| `CRAWL_2_REPORT.md` | Crawl 1 vs crawl 2 comparison — 30 issues → 23 issues, grade B- → A-, 7 issues fully eliminated. |
| `manifest.yaml` | Machine-readable audit manifest — both crawls, all commits, all phases, indexing push results. |

## How to use these

1. Read the two teaching docs in the repo root first — they're adapted for mos2es.com
2. Use these reference plans when you need the full detail or want to see exactly what we did
3. The Screaming Frog crawl reports are real exports from signalaf.com — use them as a template for what to export when you crawl mos2es.com
